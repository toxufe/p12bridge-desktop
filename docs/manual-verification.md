# P12Bridge Manual Verification Checklist

This checklist is for the remaining verification work that cannot be proven by synthetic tests alone.

## Status

- Automated local MVP walkthrough: covered by `LocalMvpWalkthroughTests`.
- Manual end-to-end walkthrough: pending.
- Real signed-IPA upload verification: pending.

Do not mark the manual walkthrough or real upload verification complete until the evidence listed here has been captured.

## Required Materials

- Windows machine with .NET 8 SDK.
- P12Bridge Desktop built from the current `main` branch.
- Paid Apple Developer account.
- Apple-issued `.cer` created from the CSR generated by P12Bridge.
- App Store distribution `.mobileprovision` for the target Bundle ID.
- Already signed `.ipa` for the same Bundle ID and Team ID.
- Existing `AppStoreInfo.plist` exported with the signed IPA package.
- Apple Transporter executable path.
- Upload credential: Apple ID plus app-specific password, or the supported App Store Connect credential path.

## Secret Handling

Safe evidence:

- Commit hash.
- App version/build shown by P12Bridge.
- Bundle ID and Team ID.
- Certificate/profile expiration dates.
- File names and local paths when needed for troubleshooting.
- Readiness check status and issue codes.
- Upload evidence copied from the `证据` action.
- Upload evidence saved from the `保存` action.
- Redacted upload log copied from the app.
- App Store Connect build status after upload.

Do not capture:

- Private key contents.
- `.p8` contents.
- `.p12` bytes or P12 password.
- JWTs or Authorization headers.
- Apple ID main password.
- App-specific password.
- Raw IPA bytes.
- Raw `.mobileprovision` bytes.
- Raw `AppStoreInfo.plist` contents.
- Unredacted Transporter stdout/stderr.

## Local Walkthrough

Record the commit hash before starting:

```powershell
git rev-parse --short HEAD
```

1. Start the app.
2. Open `制作证书`.
3. Create a Distribution certificate project.
4. Confirm the project directory, private key path, and CSR path are shown.
5. Copy the CSR and create an Apple certificate in Apple Developer.
6. Download the `.cer` file.
7. Import the `.cer` into the project.
8. Export P12 with a test password kept outside the evidence log.
9. Confirm the P12 path is shown and the operation history records only safe metadata.
10. Open `描述文件`.
11. Import the App Store `.mobileprovision`.
12. Confirm Bundle ID, Team ID, profile type, expiration, and certificate count.
13. Open `IPA 检查`.
14. Select the signed IPA.
15. Confirm Bundle ID, version, build, and signature indicators.
16. Confirm an existing sibling `AppStoreInfo.plist` is discovered, or select it in Settings.
17. Open `IPA 上传`.
18. Run upload readiness.
19. Confirm status is `可上传` or document each warning/blocking issue.
20. Click `证据` or `保存` and store the output as walkthrough evidence.

## Transporter Verify

1. Open `设置`.
2. Select the Transporter executable.
3. Select the signed IPA path.
4. Select the `AppStoreInfo.plist` path.
5. Choose the intended credential mode.
6. Enter credentials without saving them unless credential persistence is explicitly being tested.
7. Run environment validation.
8. Copy the environment result and store only the redacted output.
9. Open `IPA 上传`.
10. Run `校验`.
11. Confirm the result is `校验完成`, or store the redacted failure log and issue code.
12. Click `证据` or `保存` and store the output as verify evidence.

## Real Upload Attempt

Only run this when the IPA is intended to be delivered to App Store Connect/TestFlight.

1. Re-run upload readiness and confirm there are no blocking issues.
2. Run `上传`.
3. Confirm the app shows `待核验` after a successful upload attempt.
4. Copy the redacted upload log.
5. Open App Store Connect.
6. Check the matching app, Bundle ID, version, and build number.
7. Confirm whether the build appears, is processing, or failed processing.
8. Run `构建` in `IPA 上传`.
9. Click `证据` or `保存` and record the App Store Connect build state without copying credentials or private payloads.

## Completion Rules

- Manual end-to-end walkthrough is complete only when every local walkthrough step succeeds or has a documented product issue.
- Real upload verification is complete only when a signed IPA upload attempt has matching App Store Connect evidence.
- A failed real upload can close the verification attempt only when the failure is redacted, reproducible, and mapped to the next fix.
